# ADR-04.005: Use-Case Dependency Contracts Stay Private

## Status

Accepted at 2026-06-21

## Context

Use-cases in this codebase follow a factory pattern where an internal
`Dependencies` type describes the injected collaborators.

When that type is exported by accident, it becomes part of the public API of the
file and makes refactoring harder. The type is an implementation detail, not a
contract that other modules should depend on.

## Decision

- `Dependencies` types inside use-case implementation files must remain local to
  the file.
- Exporting `interface Dependencies`, `type Dependencies`, or re-exporting that
  symbol from a use-case implementation file is a violation.
- The rule applies to implementation files under `use-cases/`.

## Consequences

- Use-case factories keep a smaller public surface.
- Refactoring collaborators does not leak internal structure.
- The architecture reports can detect accidental API exposure.
