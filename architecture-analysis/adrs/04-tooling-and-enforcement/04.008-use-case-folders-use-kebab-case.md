# ADR-04.008: Use-Case Folders Use Kebab-Case Action Names

## Status

Accepted at 2026-06-21

## Context

Use-case folders are a primary navigation surface in the backend. When their
names are inconsistent, the tree becomes harder to scan and harder to automate
against.

A stable kebab-case convention makes use-case paths predictable and keeps the
folder name aligned with the action it represents.

## Decision

- Every use-case folder under `commands/` or `queries/` must use kebab-case.
- Folder names should describe the action performed by the use-case.
- Each folder continues to contain the `execute.ts` factory and the matching
  `execute.test.ts` file.

## Consequences

- Use-case paths become more readable and uniform.
- The architecture metrics can validate naming consistently.
- Folder names are easier to guess, grep, and review.
